# 基于协同过滤算法的智能旅游景点推荐系统 - 毕业论文完整项目文档

## 项目概述

### 项目背景与意义

随着互联网技术的快速发展和旅游业的蓬勃兴起，在线旅游平台已成为人们规划旅行的重要工具。然而，面对海量的旅游信息，用户往往难以快速找到符合自己兴趣的景点。传统的搜索和分类浏览方式效率低下，无法满足个性化需求。

本项目是一个基于Spring Boot 3 + Vue 3的全栈旅游管理平台，核心创新在于集成了基于协同过滤算法的智能推荐系统，通过分析用户的历史行为数据（浏览、收藏、购票等），为用户提供个性化的景点推荐服务，有效解决了信息过载问题，提升了用户体验和平台价值。

### 核心特性
- **智能推荐引擎**：基于用户协同过滤算法（User-Based Collaborative Filtering），实现个性化景点推荐
- **完整业务流程**：在线门票预订与支付系统，涵盖订单创建、支付、退款全流程
- **社交互动功能**：旅游笔记分享社区，支持评论、收藏等互动
- **国际化支持**：全面支持俄语界面，适应国际化需求
- **响应式设计**：支持PC端和移动端访问
- **数据可视化**：管理后台提供订单统计、用户分析等数据图表

### 技术创新点
1. **实时推荐算法**：采用实时计算策略，无需预计算和缓存，降低系统复杂度
2. **混合推荐策略**：结合协同过滤推荐（50%）和热门景点补充（50%），提高推荐质量
3. **多维度评分机制**：综合用户收藏（3分）和购票（5分）行为计算景点评分
4. **余弦相似度算法**：使用余弦相似度计算用户间相似性，提高推荐准确度
5. **前后端分离架构**：采用RESTful API设计，提高系统可扩展性和维护性

### 2.5 推荐系统模块

#### 2.5.1 推荐算法
采用**基于用户的协同过滤算法** (User-Based Collaborative Filtering)

**核心思想**：
- 找到与当前用户兴趣相似的其他用户
- 推荐这些相似用户喜欢但当前用户未接触的景点

**评分计算规则**：
- 收藏景点：+3分
- 购买门票：+5分

#### 2.5.2 相似度计算
使用**余弦相似度** (Cosine Similarity)

```java
similarity = (A · B) / (||A|| × ||B||)
```

其中：
- A, B 是两个用户的评分向量
- A · B 是向量点积
- ||A||, ||B|| 是向量的模

#### 2.5.3 推荐策略
- 50%协同过滤推荐（基于相似用户）
- 50%热门景点补充（按收藏数+评论数排序）
- 过滤已交互景点
- 实时计算，无需预计算

### 3.3 视图设计

#### 3.3.1 用户景点评分视图 (v_user_scenic_score)

用于推荐算法的核心视图，聚合用户对景点的评分。

```sql
CREATE VIEW v_user_scenic_score AS
SELECT 
    user_id,
    scenic_id,
    SUM(score) AS total_score,
    MAX(action_time) AS last_action_time
FROM (
    -- 收藏行为：3分
    SELECT 
        user_id,
        association_id AS scenic_id,
        3.0 AS score,
        create_time AS action_time
    FROM favorite
    WHERE type_code = 'SCENIC'
    
    UNION ALL
    
    -- 购买行为：5分
    SELECT 
        user_id,
        scenic_id,
        5.0 AS score,
        create_time AS action_time
    FROM order_info
    WHERE status IN ('PAID', 'COMPLETED')
) t
GROUP BY user_id, scenic_id;
```

**字段说明**：
- `user_id`: 用户ID
- `scenic_id`: 景点ID
- `total_score`: 总评分（收藏+购买）
- `last_action_time`: 最后交互时间
