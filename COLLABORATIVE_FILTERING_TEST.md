## 问题诊断

你发现每个用户看到的推荐都一样，这通常是因为：

1. **数据不足**：数据库中用户交互数据太少
2. **热门推荐覆盖**：当协同过滤无法生成推荐时，会返回热门景点
3. **相似度阈值过高**：找不到相似用户

## 算法原理

### 用户相似度计算（余弦相似度）

```
用户A向量：[景点1:3, 景点2:3, 景点3:3]
用户B向量：[景点1:3, 景点2:3, 景点4:3, 景点5:3]

共同景点：[景点1, 景点2]
点积 = 3*3 + 3*3 = 18
模A = √(3²+3²+3²) = 5.196
模B = √(3²+3²+3²+3²) = 6.0
相似度 = 18 / (5.196 * 6.0) = 0.577
```

### 推荐分数计算

```
推荐分数 = Σ(相似度 × 评分)

例如：景点4的推荐分数
= 用户2的相似度(0.577) × 用户2对景点4的评分(3.0)
= 1.731
```

## 评分权重

- **收藏**：3分
- **已支付订单**：5分
- **已完成订单**：5分

## 常见问题

### Q1: 为什么所有用户看到的推荐都一样？
A: 数据不足，所有用户都降级到热门推荐。运行 `add_test_interactions.bat` 添加测试数据。

### Q2: 如何确认协同过滤在工作？
A: 查看后端日志，应该看到"✓ 协同过滤推荐完成"而不是"⚠️ 返回热门景点"。

### Q3: 推荐结果不够个性化？
A: 需要更多用户交互数据。鼓励用户收藏和购买门票。

### Q4: 新用户怎么办？
A: 新用户没有交互历史，会自动降级到热门推荐（冷启动问题）。

## 下一步优化建议

1. **混合推荐**：协同过滤 + 基于内容（景点类别、标签）
2. **时间衰减**：最近的交互权重更高
3. **负反馈**：考虑用户不喜欢的内容
4. **缓存优化**：缓存相似度矩阵，避免每次实时计算
5. **A/B测试**：对比协同过滤和热门推荐的效果
