# 协同过滤测试 - 快速验证

## 当前状态

已禁用热门推荐降级逻辑，现在：
- ✅ 如果协同过滤工作 → 显示个性化推荐
- ❌ 如果协同过滤不工作 → 显示空列表（不显示任何景点）

这样可以清楚看到协同过滤是否真的在运行。

## 立即测试步骤

### 1. 添加测试数据（必须）
```bash
add_test_interactions.bat
```

### 2. 重启后端服务
确保代码更改生效

### 3. 测试场景

#### 场景A：用户1登录
**预期**：
- 如果有推荐 → 显示景点4,5（来自相似用户2）
- 如果空列表 → 说明协同过滤没工作

#### 场景B：用户2登录
**预期**：
- 如果有推荐 → 显示景点3,6（来自相似用户1和3）
- 如果空列表 → 说明协同过滤没工作

#### 场景C：新用户登录
**预期**：
- 空列表（因为没有交互历史）

### 4. 查看日志

**协同过滤工作的日志**：
```

✓ 用户1的交互历史（共3条）

**协同过滤不工作的日志**：
```

或
```

```

### 5. 前端控制台

**成功**：
```
========== 推荐接口返回 ==========
✓ 使用推荐结果: 2 个景点
推荐景点列表: ID:4 景点名称, ID:5 景点名称
```

**失败**：
```
========== 推荐接口返回 ==========
✗ 推荐结果为空
可能原因：
  1. 用户未登录
  2. 用户没有交互历史
  3. 找不到相似用户
```

## 可能的问题

### 问题1：所有用户都返回空列表

**原因**：数据库中没有用户交互数据

**解决**：
```bash
# 检查数据
check_recommendation_data.bat

# 添加测试数据
add_test_interactions.bat
```

### 问题2：视图不存在

**错误**：`Table 'tourism_db.v_user_scenic_score' doesn't exist`

**解决**：
```sql
-- 运行数据库初始化脚本
source sql (1)/tourism_db_routines.sql
```

### 问题3：用户ID不存在

**错误**：测试数据中的用户ID在数据库中不存在

**解决**：
修改 `add_test_interactions.sql`，使用实际存在的用户ID

### 问题4：景点ID不存在

**错误**：测试数据中的景点ID在数据库中不存在

**解决**：
修改 `add_test_interactions.sql`，使用实际存在的景点ID
