## 为什么图片显示"图片加载失败"？

新添加的景点数据中，`cover_image` 字段设置为 `NULL`，这样前端会显示占位图。这是正常的，因为：

1. 我们没有实际的景点图片文件
2. 使用 NULL 比使用不存在的路径更好
3. 前端已经有占位图的处理逻辑

## 如何添加真实图片？

### 方法1：通过管理后台上传（推荐）

1. 以管理员身份登录（admin/admin123）
2. 进入"景点信息管理"
3. 点击"编辑"按钮
4. 上传景点封面图片
5. 保存

### 方法2：使用现有图片URL

如果你有图片URL，可以直接更新数据库：

```sql
UPDATE scenic_info 
SET cover_image = 'http://localhost:1000/file/your-image-file.jpg' 
WHERE id = 景点ID;
```

### 方法3：批量使用占位图服务

使用在线占位图服务（如 placeholder.com）：

```sql
UPDATE scenic_info 
SET cover_image = CONCAT('https://via.placeholder.com/400x300?text=', name)
WHERE cover_image IS NULL;
```
